1. Introduction

Overview

The multcomp package in R is designed for simultaneous inference in linear models, specifically focused on multiple comparisons. It provides tools to test hypotheses about multiple parameters in a model while controlling for family-wise error rates.

Use Cases

  • Comparing group means in ANOVA.
  • Simultaneous confidence intervals.
  • Adjusting p-values for multiple testing.

Benefits

  • Handles a variety of multiple comparison tests.
  • Provides flexibility with user-defined contrasts.
  • Supports many types of linear models, including generalized linear models and mixed-effects models.

2. Getting Started

Loading the Package

# Load the multcomp package
pacman::p_load(multcomp)

Basic Usage

Let’s start with a simple example:

# Example dataset
data(warpbreaks)

# Fit a linear model
model <- lm(breaks ~ wool + tension, data = warpbreaks)

# Perform multiple comparisons
comparison <- glht(model, linfct = mcp(tension = "Tukey"))

# Summarize the results
summary(comparison)

---
Signif. codes:  0 '***' 0.001 '**' 0.01 '*' 0.05 '.' 0.1 ' ' 1
(Adjusted p values reported -- single-step method)

Explanation

  1. mcp: Specifies the type of multiple comparisons (e.g., Tukey’s test).
  2. glht: General linear hypothesis testing function.
  3. summary: Displays adjusted p-values and confidence intervals.

3. Key Functions and Features

glht()

Description: Performs multiple comparisons or general linear hypothesis tests.

Syntax:

multcomp::glht(model, linfct = mcp("factor"="method"))

Arguments:

  • model: A fitted linear model object.
  • linfct: Defines the hypotheses to test (e.g., Tukey or user-defined contrasts).

Example:

comparison <- glht(model, linfct = mcp(tension = "Tukey"))
summary(comparison)

mcp()

Description: Creates predefined contrasts for a factor.

Syntax:

mcp(factor = "method")

Example:

linfct = mcp(tension = "Tukey")

5. Advanced Features

Custom Contrasts

Create user-defined contrasts:

contrast_matrix <- rbind(
    "M - L" = c(-1, 1, 0),
    "H - L" = c(-1, 0, 1),
    "H - M" = c(0, -1, 1)
)
comparison <- glht(model, linfct = contrast_matrix)
summary(comparison)

6. Troubleshooting and FAQs

Common Issues

  1. Error: Non-conformable arguments

    • Ensure the linfct matches the model’s factor levels.

  2. Confusion with model types

    • Use glht with models that support linear hypotheses (e.g., lm, lmer).

FAQs

  • Can multcomp handle unbalanced data? Yes, but ensure your model accounts for this.

7. Best Practices

  • Always check model diagnostics before multiple comparisons.
  • Use visualizations like plot() to complement numerical results.
  • For large datasets, consider summarizing key results to avoid overloading the output.
