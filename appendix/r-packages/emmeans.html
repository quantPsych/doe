1. Introduction

The emmeans package in R simplifies post-hoc analysis and estimation of marginal means from statistical models. It provides tools to estimate, compare, and test means across levels of predictors while accounting for the model structure.

Overview

  • Purpose: Provides estimated marginal means (EMMs) or least-squares means.
  • Core Features:
    • Compute EMMs for any combination of factors.
    • Perform pairwise comparisons or custom contrasts.
    • Integrate with popular visualization packages like ggplot2.
    • Easily handle model interactions and transformations.
  • Key Supported Models:
    • Linear models (lm, aov).
    • Generalized linear models (glm).
    • Mixed-effects models (lme4, nlme).

What Are Marginal Means?

Marginal means are predicted values from a model, adjusted to account for the distribution of covariates or other factors. They provide a clearer interpretation of group effects compared to raw means.

2. Getting Started

Loading the Package

library(emmeans)

Example Dataset

We will use the warpbreaks dataset, which contains data on the number of breaks in yarn (response variable breaks) across two factors: wool type (wool) and tension level (tension).

Basic Model

# Fitting a linear model
lm_model <- lm(breaks ~ wool * tension, data = warpbreaks)

Basic Marginal Means

# Marginal means by wool
emmeans(lm_model, ~ wool)
 wool emmean   SE df lower.CL upper.CL
 A      31.0 2.11 48     26.8     35.3
 B      25.3 2.11 48     21.0     29.5

Results are averaged over the levels of: tension 
Confidence level used: 0.95

3. Key Functions in emmeans

3.1 emmeans()

The emmeans() function is the foundation of the package, used to estimate marginal means.

Usage:

emmeans(object, specs, by = NULL, options = NULL, ...)

Arguments:

  • object: A fitted model (e.g., lm, glm, or lmer).
  • specs: Formula specifying factors for which marginal means are computed. Use | to specify conditions (e.g., ~ wool | tension).
  • by: Variables to group results.
  • options: Adjustments like type of prediction (response, link, etc.).

Examples:

# Marginal means for wool
emmeans(lm_model, ~wool)
 wool emmean   SE df lower.CL upper.CL
 A      31.0 2.11 48     26.8     35.3
 B      25.3 2.11 48     21.0     29.5

Results are averaged over the levels of: tension 
Confidence level used: 0.95 
# Marginal means of tension within each wool group
emmeans(lm_model, ~ tension | wool)
wool = A:

 M         24.0 3.65 48     16.7     31.3
 H         24.6 3.65 48     17.2     31.9

wool = B:

3.2 contrast()

The contrast() function performs post-hoc comparisons between marginal means, such as pairwise comparisons or custom contrasts.

Usage:

contrast(object, method = "pairwise", interaction = FALSE, by = NULL, ...)

Arguments:

  • object: An emmGrid object (output of emmeans()).
  • method: Type of comparison. Common methods:
    • "pairwise": Pairwise differences.
    • "poly": Polynomial contrasts.
    • "eff": Effect contrasts (deviation from overall mean).
  • interaction: Defines interaction contrasts, generating contrasts of contrasts for multi-factor interactions.
  • by: Grouping variables for separate contrasts.

Examples:

# Pairwise comparisons of tension levels
warp.emm <- emmeans(lm_model, ~ tension | wool)
contrast(warp.emm, method = "pairwise")

3.3 pairs()

This function simplifies pairwise comparisons (shortcut for contrast(…, method = "pairwise")).

Usage:

pairs(object, reverse = FALSE, ...)

Arguments:

  • reverse: If TRUE, reverses the order of comparisons.

pairs(warp.emm)

4. Advanced Functionality

4.1 Interaction Contrasts

Interaction contrasts allow you to compare interactions between factors. The interaction argument in contrast() enables this.

Example:

# Interaction contrast for wool and tension
interaction_contrast <- contrast(warp.emm, interaction = c(wool = "eff", tension = "poly"))
interaction_contrast
wool = A:

 M effect       -7.04 2.98 48  -2.363  0.0222
 H effect       -6.48 2.98 48  -2.177  0.0344

wool = B:
 tension_eff estimate   SE df t.ratio p.value
 L effect        2.96 2.98 48   0.995  0.3247
 M effect        3.52 2.98 48   1.182  0.2432
 H effect       -6.48 2.98 48  -2.177  0.0344
coef(interaction_contrast)  # View coefficients

4.2 Scaling and Offsetting

Use scale and offset to transform results for interpretation (e.g., converting units).

Example:

# Convert temperature from Celsius to Fahrenheit
mod <- lm(Water.Temp ~ poly(stack.loss, degree = 2), data = stackloss)
emm <- emmeans(mod, "stack.loss", at = list(stack.loss = 10 * (1:4)))
contrast(emm, "identity", scale = 9 / 5, offset = 32)
 contrast     estimate    SE df t.ratio p.value
 stack.loss10     65.9 0.833 18  79.138  <.0001
 stack.loss20     72.1 1.020 18  71.059  <.0001
 stack.loss30     76.8 1.160 18  66.067  <.0001
 stack.loss40     80.0 1.720 18  46.389  <.0001

5. Visualizing Results

Interaction Plots

The emmip() function creates interaction plots to visualize EMMs:

# Interaction plot for wool and tension
library(ggplot2)
emmip(warp.emm, wool ~ tension)

Pairwise Comparisons Plot

# Pairwise comparison plot
plot(pairs(warp.emm))

6. Common Issues and Troubleshooting

Common Issues

  1. Degrees of freedom not available:

    • Some models (e.g., Bayesian) require manual specification of degrees of freedom.

  2. Invalid contrast method:

    • Ensure the method matches the structure of your emmGrid object.

Best Practices

  • Always inspect the emmGrid object using summary() before applying contrasts.
  • Use confint() to obtain confidence intervals.
  • Apply adjust for proper corrections in multiple comparisons.

7. Detailed Argument Reference
